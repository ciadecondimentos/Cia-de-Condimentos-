🧪 Teste de Responsividade - Painel Admin

Use este guia para validar a responsividade da interface em diferentes tamanhos de tela.

→ Abrir Admin Panel

📱 Breakpoints para Testar

Mobile Small: 320px - 480px

Dispositivos: Galaxy S5, iPhone SE, iPhone 5

  • Menu hamburger visível (☰)
  • Sidebar como drawer (hidden by default)
  • Topbar em 2 linhas
  • Stats cards em 1 coluna
  • Tabelas em card-style
  • Botões full-width
  • Sem scroll horizontal
  • Font size inputs: 16px
Mobile: 480px - 767px

Dispositivos: iPhone 12, Samsung S10, Moto G

  • Menu hamburger still visible
  • Sidebar drawer ainda escondido
  • Conteúdo padding: 12px
  • Forms com 1 coluna
  • Modais bottom-sheet
  • Tabelas card-style mantidas
Tablet: 768px - 1023px

Dispositivos: iPad, iPad Mini, Tablets Android

  • Sidebar visível (fixed 280px)
  • Menu hamburger hidden
  • Stats grid: 2 colunas
  • Dashboard grid: 2 colunas
  • Tabelas normal ou card-style
  • Content padding: 32px
Desktop: 1024px+

Dispositivos: Laptops, Monitors

  • Sidebar 280px fixo à esquerda
  • Stats grid: 4 colunas
  • Dashboard grid: 2 colunas
  • Tabelas completas
  • Layout desktop normal
  • Todas features visíveis

🎯 Funcionalidades a Validar

Menu e Navegação
  • Botão hamburger abre/fecha sidebar em mobile
  • Overlay semi-transparente aparece
  • Clicar no overlay fecha sidebar
  • Selecionar página fecha sidebar automaticamente
  • Página ativa é destaque no menu
Dashboard
  • Stats cards responsivos
  • Cards em grid adaptivo
  • Bar chart visível e escalado
  • Period tabs funcionam
  • Dados carregam corretamente
Tabelas (Produtos, Pedidos, CRM)
  • Search box funciona em mobile
  • Filters (selects) full-width em mobile
  • Em mobile: tabela convertida para cards
  • Labels (data-label) aparecem
  • Botões ação clicáveis (44px+)
  • Sem scroll horizontal
  • Botão "Novo" full-width em mobile
Formulários
  • Inputs full-width em mobile
  • Selects full-width em mobile
  • Textareas full-width em mobile
  • Font-size inputs: 16px (anti-zoom)
  • Min-height inputs: 44px
  • Botões full-width em mobile
  • Form-row-2 → 1 col em mobile
  • Form-row-3 → 1 col em mobile
Modais
  • Desktop: modal centered
  • Mobile: modal bottom-sheet
  • Botão close funciona
  • Overlay fecha modal
  • Conteúdo scrollável se necessário
  • Buttons empilhados em mobile

📊 Exemplo de Testes

Teste 1: Galaxy S5 (360×640)

1. Abrir admin.html em DevTools
2. Selecionar Galaxy S5
3. Verificar se menu hamburger aparece
4. Clicar hamburger → sidebar deve aparecer
5. Clicar overlay → sidebar deve desaparecer
6. Navegar para Produtos → sidebar fecha
7. Verificar tabela em card-style
8. Tentar preencher formulário → fácil?
9. Verificar scroll (não deve haver horizontal)

Teste 2: iPhone 12 (390×844)

1. Testar em portait e landscape
2. Verificar dashboard stats (1 coluna)
3. Testar cards e charts
4. Abrir um modal → deve ser bottom-sheet
5. Preencher e submit formulário
6. Verificar toast notification

Teste 3: iPad (768×1024)

1. Sidebar deve estar visível (não drawer)
2. Stats grid: 2 colunas
3. Dashboard: 2 colunas
4. Tabelas com mais espaço
5. Modais centered (não bottom-sheet)

✅ Conclusão

Se todos os testes passarem, o painel administrativo está 100% responsivo!
